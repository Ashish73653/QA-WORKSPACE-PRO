# Requirement Traceability Matrix (RTM)

Use this matrix to map SRS requirements to implementation and verification evidence.

## Columns

- Req ID
- SRS Section
- Requirement Statement
- Type (Functional/Non-Functional/Integration/Security)
- Priority (P0/P1/P2)
- Acceptance Criteria
- Design Artifact
- Implementation Artifact
- Test Case IDs
- Test Status
- Release Version
- Notes

## Guidance

1. Every implementable SRS statement must get a Req ID.
2. Each Req ID must map to at least one test case.
3. A requirement is release-ready only when test evidence is attached.
